Git在本地新建仓库并提交到Github

By vivi

发表于 2015-08-25

文章目录

Github作为当前使用最广的开源代码库以及版本控制系统，在码农圈受到的极大欢迎。
本人作为一只小白，虽然很早就了解github，但并未在实际开发中真正使用上。这篇文章只是一次实践尝试，通过git命令在本地新建仓库并提交到github远程服务器。


创建本地仓库

仓库，又名版本库，英文名repository，可以理解成我们项目所在的目录，这个目录下面的所有文件都可以被Git管理，每个文件的修改、删除，Git都可以追踪历史。

首先创建一个本地项目，如下结构

# ls -l
total 16
drwxr-xr-x 2 root root 4096 Aug 25 03:03 log
-rwxr-xr-x 1 root root 8540 Aug 18 07:44 transportmail.py

为本地项目创建仓库，进入项目目录，执行

[root@bestvivi test]# git init
Initialized empty Git repository in /root/test/.git/

git init只是初始化了仓库(会在当前目录下生成一个.git隐藏文件)，但还并未将项目文件提交到仓库，接下来执行

# git add .
# git commit -m "first commit"
[master (root-commit) a2108e0] first commit
 1 file changed, 257 insertions(+)
 create mode 100755 transportmail.py

git add是将当前目录下的项目文件添加到仓库，git commit顾名思义就是提交修改。接下来就可以将本地项目push到远程github上保存了。


提交本地项目到远程仓库

首先添加本地VPS服务器公钥认证到Github并建立安全连接，方便后续的代码提交。在Github上创建一个远程仓库并命名，用于存放本地仓库的提交。

创建远程仓库的标志

# git remote add origin git@github.com:bestvivi/mailtransport.git

其中origin是远程仓库的别名，可以随便起，默认是origin；git@github.com:bestvivi/mailtransport.git就是ssh连接到远程仓库的路径

查看添加的远程仓库

# git remote -v
origin    git@github.com:bestvivi/mailtransport.git (fetch)
origin    git@github.com:bestvivi/mailtransport.git (push)

将本地仓库push到远程仓库

# git push origin master
Counting objects: 3, done.
Compressing objects: 100% (2/2), done.
Writing objects: 100% (3/3), 2.89 KiB | 0 bytes/s, done.
Total 3 (delta 0), reused 0 (delta 0)
remote: To git@github.com:bestvivi/mailtransport.git
 * [new branch]      master -> master

其中master是本地仓库的主干分支，至此，已经完成了将本地仓库提交到Github的工作。
